# 🤖 주문형 멀티 에이전트 소스코드 생성 플랫폼 (Meta-AI Factory)

사용자가 원하는 AI 팀의 주제를 입력하면, 로컬 LLM 환경을 기반으로 맞춤형 CrewAI 멀티 에이전트 아키텍처를 설계하고 정석 파이썬 소스코드를 동적으로 빌드해 주는 캡스톤 디자인 졸업작품 프로젝트입니다.

본 저장소는 협업 표준 규격을 준수하여 환경 종속적인 무거운 폴더(`.venv`, `node_modules`)를 제외한 **순수 소스코드 원본**만 가볍게 포함하고 있습니다. 최초 다운로드 시 아래 가이드에 따라 의존성 패키지를 빌드한 후 구동해 주시기 바랍니다.

---

## 🏛️ 시스템 아키텍처 개요
* **Frontend**: React + TypeScript (Vite)
* **Backend**: FastAPI (Uvicorn), Async Pydantic Pipeline
* **Orchestrator Engine**: CrewAI (최신 비동기 `kickoff_async` 아키텍처)
* **Local LLM**: Ollama (Qwen2.5-Coder-14B 모델)

---

## 🚀 단계별 실행 및 가동 가이드

### 1단계: 로컬 LLM (Ollama) 환경 세팅
본 프로젝트는 외부 API Key 없이 **100% 로컬 GPU 인프라** 환경에서 작동합니다. (최소 VRAM 8GB 이상 권장)

1. [Ollama 공식 홈페이지](https://ollama.com/)에서 OS에 맞는 설치 파일을 다운로드하여 설치합니다.
2. 터미널(CMD 또는 PowerShell)을 열고 Ollama가 정상 설치되었는지 확인합니다:
   ```bash
   ollama --version
에이전트 설계 및 코드 생성을 담당할 14B 코드 모델을 다운로드하고 백그라운드에 가동시킵니다:

ollama run qwen2.5-coder:14b
Note: 최초 실행 시 약 9GB의 모델 다운로드가 진행됩니다. 터미널 창에 >>> 입력 대기 상태가 뜨면 준비가 완료된 것이며, /exit로 빠져나올 수 있습니다.

### 2단계: 파이썬 가상환경 및 백엔드 의존성 복구
프로젝트 루트 폴더(capstone_ai_team/)에서 파이썬 가상환경을 생성하고 활성화합니다:

# 가상환경 생성 및 활성화 (Windows 기준)
python -m venv .venv
.venv\Scripts\activate
배포 명세서(requirements.txt)를 이용해 필수 패키지를 일괄 설치합니다:

pip install -r requirements.txt

### 3단계: 통합 환경 빌드 및 원클릭 동시 구동 (npm start)
본 프로젝트는 효율적인 개발 환경을 위해 백엔드와 프론트엔드를 터미널 한 창에서 동시에 제어하는 통합 구동 자동화 아키텍처가 설계되어 있습니다.

파이참 터미널 창(루트 폴더 경로)에서 프로젝트를 구동하기 위한 모듈들을 최초 1회 다운로드합니다. (실행 중인 OS 환경에 최적화된 깨끗한 node_modules가 자동 생성됩니다.)

npm install
패키지 빌드가 끝나면, 아래의 통합 단축 명령어 단 한 줄로 백엔드와 프론트엔드를 동시에 가동합니다:

npm start
터미널 하나에 FastAPI 서버와 리액트 Vite 서버 개발 로그가 동시에 출력되는 것을 확인하고, 브라우저에서 http://localhost:5173에 접속합니다.

종료 방법: 구동 중인 터미널 창에서 Ctrl + C를 누르면 두 서버가 안전하게 동시 종료됩니다.

### 4단계: 최종 통합 테스트 흐름
1. 리액트 대시보드 화면 인풋창에 유저 맞춤형 요구사항을 입력합니다.
    예시: “주식 시장 데이터를 분석하고 리포트를 써주는 금융 투자 AI 팀”
2. [AI 크루 주문하기] 버튼을 클릭합니다.
3. 약 30~40초 후 엄격한 아키텍처 지침이 반영된 순수 CrewAI 파이썬 소스코드가 화면 하단에 왼쪽 정렬되어 깔끔하게 출력됩니다.
4. 우측 상단에 동적으로 나타나는 [📋 코드 복사] 및 [💾 파일 다운로드 (.py)] 기능을 통해 소스코드를 추출하여 즉시 배포 및 테스트가 가능합니다.
